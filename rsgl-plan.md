## Phase 2: Rename Package (sglref → rsgl)

Systematic rename of the package name across all files. Every occurrence
of `sglref` must become `rsgl`.

### 2.1 DESCRIPTION

- Change `Package: sglref` → `Package: rsgl`

- Change `Title:` to something like
  `An Implementation of the SGL Graphics Language`

  - CRAN rule: title must be in Title Case, must not redundantly start
    with package name or “A package for…”

- Change `Description:` to a proper paragraph (at least one sentence):

      Parses and executes SGL (Structured Graphics Language) statements against
      'DuckDB' databases to produce 'ggplot2' visualizations. SGL is a declarative
      language for generating statistical graphics from relational data. See
      Chapman (2025) <doi:10.48550/arXiv.2505.14690> for the language specification.

- Fix typo: `"packagee"` → `"package"` (in the description text above
  this won’t be present, but verify)

- Change `URL:` to `https://github.com/sgl-projects/rsgl`

- Change `BugReports:` to `https://github.com/sgl-projects/rsgl/issues`

- Fix inconsistent indentation (sglref mixes tabs and spaces) — use
  spaces only

- Consider updating version to `0.1.0` (first public release)

### 2.2 NAMESPACE

Change:

    useDynLib(sglref, .registration = TRUE)

to:

    useDynLib(rsgl, .registration = TRUE)

Note: The NAMESPACE is generated by roxygen2, so this change must come
from updating the source roxygen comments and re-running
`roxygen2::roxygenise()`. But since we may not have R available, we can
update both the source comment and the generated NAMESPACE directly.

### 2.3 R/sglref-package.R → R/rsgl-package.R

- Rename the file

- Change `@useDynLib sglref` → `@useDynLib rsgl`

- File content should be:

## Phase 3: CRAN Compliance Fixes

### 3.1 Makevars cleanup

The current `src/Makevars` links LAPACK/BLAS/FLIBS, which are
unnecessary for a text parser. Update to:

``` makefile
PKG_CFLAGS = $(SHLIB_OPENMP_CFLAGS)
PKG_CXXFLAGS = $(SHLIB_OPENMP_CXXFLAGS)

OBJECTS = RcppExports.o sgl_to_rgs.o parser.tab.o scanner.o aes.o geom.o cta.o qual.o scale.o keyword.o

all: $(SHLIB)

parser.tab.o: parser.tab.c parser.tab.h
scanner.o: scanner.c parser.tab.h
```

Actually, since SGL doesn’t use OpenMP either, simplify further:

``` makefile
OBJECTS = RcppExports.o sgl_to_rgs.o parser.tab.o scanner.o aes.o geom.o cta.o qual.o scale.o keyword.o

all: $(SHLIB)

parser.tab.o: parser.tab.c parser.tab.h
scanner.o: scanner.c parser.tab.h
```

### 3.2 Makevars.win

Create `src/Makevars.win` for Windows portability:

``` makefile
OBJECTS = RcppExports.o sgl_to_rgs.o parser.tab.o scanner.o aes.o geom.o cta.o qual.o scale.o keyword.o

all: $(SHLIB)

parser.tab.o: parser.tab.c parser.tab.h
scanner.o: scanner.c parser.tab.h
```

### 3.3 flex_overrides.h review

The file redefines `exit()`, `stdout`, and `stderr` to prevent the
flex-generated scanner from calling system exit or writing to
stdout/stderr (which is forbidden in R packages). These overrides are
necessary and appropriate. CRAN may flag them in a NOTE but they are
standard practice for packages embedding flex/bison output. No change
needed but be aware.

### 3.4 .Rbuildignore updates

Update to remove references to files we didn’t copy, and add any new
exclusions:

    ^LICENSE\.md$
    ^README\.Rmd$
    ^\.github$
    ^doc$
    ^Meta$
    ^\.claude$
    ^rsgl-plan\.md$

Removed entries for `makefile`, `src/test`, `src/scanner.l`,
`src/parser.y` since those files don’t exist in rsgl.

### 3.5 Compiler warnings

The flex/bison generated code may produce compiler warnings (e.g.,
unused functions like `yyunput`, `input`). These are common and can be
suppressed if needed. If R CMD check produces WARNINGs about them, add
to Makevars:

``` makefile
PKG_CFLAGS = -Wno-unused-function
```

Only add this if warnings appear during testing.

### 3.6 CRAN DESCRIPTION requirements

Ensure: - `Authors@R` uses proper ORCID if available - `Description`
field is at least one complete sentence, ending with a period - Software
names in Description are single-quoted (‘DuckDB’, ‘ggplot2’) - URLs/DOIs
are properly formatted - No trailing whitespace or mixed indentation

## Key Risks and Mitigations

| Risk                                                      | Mitigation                                                                                  |
|-----------------------------------------------------------|---------------------------------------------------------------------------------------------|
| Rcpp symbol naming breaks after rename                    | Carefully update all `_sglref_` → `_rsgl_` in RcppExports.cpp, RcppExports.R, and NAMESPACE |
| Flex/bison generated code produces compiler warnings      | Suppress with `-Wno-unused-function` in Makevars if needed                                  |
| Windows compilation fails (no Makevars.win)               | Create Makevars.win with same content as Makevars                                           |
| CRAN NOTE about LAPACK/BLAS linkage                       | Remove unnecessary `$(LAPACK_LIBS) $(BLAS_LIBS) $(FLIBS)` from Makevars                     |
| Visual snapshot tests fail on different platforms         | Accept that vdiffr tests may need platform-specific snapshots                               |
| CRAN NOTE about `exit()` redefinition in flex_overrides.h | This is standard practice; document in cran-comments.md if submitting                       |
